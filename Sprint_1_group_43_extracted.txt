What  could  the  Co-op  Support  App  need?   User  must  create  an  account  ●  sign  up  with  email,  username,  password  ●  can  then  login  with  selected  credentials  ●  based  on  account  roles,  different  people  can  have  different  features  in  the  program  ●  if  password  forgotten,  reset  it  ●  have  2FA  if  desired  (optional)  ●  depending  on  desired  role,  user  will  have  to  submit  documents  to  be  assigned  their  role   Students/applicants  can  use  it  to:  ●  upload  their  resumes,  cover  letters,  transcripts,  and  other  information  ●  enter  their  institution,  degree,  program,  grades  ●  choose  their  type  of  co-op  work  they  want  to  do  ●  apply  to  job  postings  ●  view  applications  statuses  (pending,  accepted,  rejected)  ●  search  and  filter  for  jobs  ●  view  application  deadlines  and  requirements  ●  update  any  changes  to  resume  ●  report  their  co-op  progress  after  completing  their  internship  ●  view  past  applications   Features  for  recruiters:  ●  can  submit  and  manage  job  postings  ●  review  applications  from  students  ●  can  accept  or  reject  the  application  from  students   Features  for  co  op  coordinators:  ●  manage  student  profiles   ●  supervise  job  postings  ●  supervise  student  activity  ●  be  able  to  communicate  with  students  on  advice  for  jobs  or  resume   App  features:  ●  based  on  previous  applications,  the  job  algorithm  will  tailor  to  user’s  preferences  ●  notify  student  on  application  deadline  alerts   ●  notify  students  on  new  job  postings

Co-op  Application  Management  System  
Students  must  be  able  to  submit  their  co-op  application  through  a  GUI  while  coordinator  requires  a  dashboard  to  review  submissions  and  assign  application  statuses,  such  as  approved  or  pending.  
The  system  must  provide  confirmation  feedback  to  students.  
High  -  Develop  student  application  submission  GUI.  
-  Implement  input  validation  for  name,  student,  ID,  and  email.  
-  Store  applications  in  the  database.  
-  Create  coordinator  dashboard  view.  
-  Implement,  approve  and  reject  pending  controls.  
-  Enable  application  status  tracking  for  students.  
9hr  3hr

Secure  Student  Portal  
Accepted  students  require  a  secure  login  to  access  restricted  reporting  features.  The  system  must  validate  credentials  against  the  database.  Once  logged  in,  students  should  only  be  able  to  see  their  own  status  and  submission  forms,  protecting  the  privacy  of  other  applicants.  
High  -  Build  an  authentication  system.  
-  Implement  login  validation  logic.  
-  Create  session  management  and  timeout  features.  
6hr  2.5hr  
-  Security:  Protect  database  from  unauthorized  modifications  .  
-  Ensure  student  names/data  are  hidden  from  other  unauthorized  users.  
Employer  Job  Posting  Portal  
Employers  require  a  portal  to  create,  edit,  and  manage  job  postings  available  to  students.  Employers  must  be  able  to  specify  job  descriptions,  qualifications,  deadlines,  and  application  criteria.  
These  postings  should  automatically  become  eligible  to  students  in  the  system.  
High  -  Develop  employer  registration  and  login  workflow.  
-  Create  job  posting  submission  forms,  implement,  edit/delete  job  posting  functionality.  
-  Store  postings  in  the  database.  
-  Enable  student  search  and  filtering  of  job  listings.  
9hr  1.5hr

Work  Term  Documentation  
Students  must  upload  their  final  work-term  reports  as  PDF  files.  The  system  needs  to  verify  the  file  type  and  check  the  current  date  against  a  set  deadline.  A  report  template  should  be  accessible  via  a  download  link  to  ensure  the  student  follows  the  required  formatting  standards.  
Med  -  Integrate  PDF  file  upload  module.  
-  Develop  deadline  comparison  logic.  
-  Add  a  static  link  for  report  template  download.  
5hr  2hr  
-  Limit  file  size  to  <  10MB.  
-  Display  "Success"  or  "Failure"  Boolean  result  upon  upload.  
-  Disable  upload  functionality  after  the  deadline  passes.  
Admin  Tracking  &  Reminders  
The  coordinator  requires  an  event  monitor  to  track  the  status  of  all  documents.  The  system  must  generate  lists  of  students  who  have  missing  reports  or  evaluations  and  allow  the  coordinator  to  send  "angry  emails"  to  those  who  have  missed  deadlines.  
 
Med  -  Create  filtered  views  (Accepted,  Rejected,  Missing  Docs).  
-  Implement  automated/manual  email  trigger  system.  
-  Develop  a  log  for  rejected  co-op  placements.  
5hr  1.5hr  
-  Reminders:  Trigger  emails  specifically  for  overdue  items  .  
-  List  results  in  chronological  order  based  on  deadline  date.
